# mysql

mysql架构 应用层：负责连接客户端，响应客户端请求，返回数据

逻辑层：包括sqk接口，解析器，优化器，cache和buffer

数据库引擎层：innodb和myisam

物理层：文件存储，日志

sql语言执行过程 1.客户端先进行身份认证和权限相关

2.执行语句会先查询缓存

3.没有命中缓存，sql语句就会经过解析器，分析语句，包括语法检查

4.通过优化器，按照mysql认为嘴优的方案去执行

5.执行语句，从存储引擎返回数据

mysql的共享锁排它锁 共享锁也称为读锁，多个用户在同一时刻读取同一资源，而且不互相干扰

排它锁也叫写锁，会阻塞其它的写锁和读锁，确保在同一时刻只给一个用户写入并防止其他用户读取这一资源

表级锁：对整张表加锁，加锁快，并发能力低

行锁：锁住某一行，如果表存在索引，记录锁是锁在索引上的，如果表没有索引，innodb会创建一个隐藏的索引加锁。 优点：行级锁会大大减少数据库操作的冲突，加锁粒度最小，并发度高 缺点：加锁开销最大，加锁慢，会出现死锁

如何解决死锁

  1. 加锁的时，给锁设置顺序，时限
  2. 放弃锁的请求，或者给几个线程设置优先级，让几个线程回退，剩下的线程会按照优先级去进行操作

乐观锁：对于数据库冲突保持一种乐观的态度，操作数据的时候不会加锁，只有提交数据时才去验证数据是否存在冲突

悲观锁：对于数据冲突保持一种悲观的态度，修改数据之前把数据锁住，然后再进行数据读写，在它释放锁之前任何人都不能对其数据进行操作，一般数据库锁是按照悲观锁机制去实现的

innodb支持事务，基于聚簇索引建立的，支持表级锁和行级锁，支持外键，适合数据增删改查频繁的情况 事务内的语句，要么全部执行成功，要么全部执行失败

  • 原子性：一个事务的所有操作，要么全部完成，要么全部不完成
  • 一致性：事务执行前后数据库状态保存一致
  • 隔离性：多个并发事务对数据库进行操作，之间互不干扰
  • 持久性：事务执行完毕，对数据修改是永久的，系统故障也不会丢失

索引是一种数据结构，关系型数据库中，索引具体是一种对数据库中一列或多列的值进行排序的存储结构

索引优点： 提高数据查询的效率，

常见索引 b-索引 哈希索引 全文索引

简述mysql优化流程

  1. 首先通过慢日志定位执行较慢的sql语句
  2. 利用explain对这些关键字段进行分析
  3. 根据分析结果进行优化

mysql主从复制

优点：读写分离，提升数据库负载性能，容灾

流程：

  1. 事务完成前，主库在binlog上记录这些改变，在binlog完成写入后，主库告知存储引擎提交事务
  2. 从库将binlog复制到对应的中继日志，将这些事件写入中继日志

redis redis数据类型；string字符串，list列表，hash，set，zset

什么情况下使用redis

  1. 针对热点数据进行缓存
  2. 针对特定西安市数据存放
  3. 分布缓存

什么数据可以放缓存？

  1. 不需要实时更新但是又极其消耗数据库的数据
  2. 需要实时更新，但是更新频率不高的数据
  3. 某个时刻访问量极大，而且更新很频繁的数据

什么数据不能放缓存？ 涉及到钱、密匙、业务关键核心数据等

redis和memecache的区别

  1. redis是单线程模型，memecach是多线程异步io
  2. redis支持数据持久化，memecach不支持
  3. redis支持数据格式更多

redis集群部署模式

  1. 主从复制
  2. 哨兵模式

redis常用命令 del set hset get hget len lset sadd

redis和mysql的区别 类型上：mysq是关系型数据库，redis是非关系型数据库 作用上：mysql用于持久化的存储数据到硬盘，但是速度较慢 redis用户存储到内存，读取速度快 redis对内存的要求高 mysql对内存要求比较高 mysql偏向于存数据，redis偏向快速取数据

mongodb、mysql、redis mysql的更新是直接写入硬盘，性能低 redis是写入内存，高性能 MongoDB是先写入内存，然后持久化到硬盘

关系型数据库和非关系型数据库
1、数据存储方式不同：关系数据库为表格形式，非关系数据库为文档或图结构

2、扩展方式不同：关系数据库可纵向扩展，可提高处理能力，非关系数据库为天然分布式，通过更多的数据服务器来分担负载

3、事务处理支持不同：关系数据库善于处理事务原子性细粒度控制，方便事务回滚，非关系数据库着重于处理大数据。
