# docker安装

在Docker官方网站下载安装文件：https://hub.docker.com/editions/community/docker-ce-desktop-mac 下载 Docker.dmg安装文件，直接双击安装完成就可以了。

登陆自己的Docker账号，可以直接在终端里使用里 检查安装成功 docker version

容器和镜像相关命令

  • docker container run < image name > 创建容器的命令
  • docker container ls 查看容器信息和状态 -a 详细信息
    • CONTAINER ID : 容器对应的ID，这个是唯一的
    • IMAGE : 使用的镜像名称，显示不同
    • COMMAND : 执行的相关命令
    • CREATED: 创建的时间
    • STATUS: 目前镜像的状态，一般会有两种状态Up和Exited.
    • PORTS: 协议和端口
    • NAMES: 容器的名称，名字是Docker随机生成的
docker container stop <name or ID>  停止容器
docker container stop  <ID1  ID2  ID3>  停止多个容器
docker container stop $(docker container ps -qa)  停止正在运行的容器
docker container rm <name or ID>  删除容器
docker container rm <ID or Image Name > -f 强制删除容器
docker container run -d -p 80:80 nginx 端口映射
docker container logs -f <ID or Image name>  查看后台日志
docker exec -it <ID or Image name> sh 进入容器的交互模式

镜像相关命令

docker image pull <Image name:版本>
docker image ls
docker image  inspect <IMAGE ID>  查看具体镜像信息
docker image rm <Image ID>
docker image save old-name:latest -o new-name.image  导出镜像
docker image load -i .\new-box.image

Dockerfile写法

FROM ubuntu:latest
RUN apt-get update && \
    apt-get install -y wget && \
    wget https://github.com/ipinfo/cli/releases/download/ipinfo-2.0.1/ipinfo_2.0.1_linux_amd64.tar.gz && \
    tar zxf ipinfo_2.0.1_linux_amd64.tar.gz && \
    mv ipinfo_2.0.1_linux_amd64 /usr/bin/ipinfo && \
    rm -rf ipinfo_2.0.1_linux_amd64.tar.gz

CMD和ENV示例

FROM ubuntu:latest
ENV VERSION=2.0.1
RUN apt-get update && \
    apt-get install -y wget && \
    wget https://github.com/ipinfo/cli/releases/download/ipinfo-${VERSION}/ipinfo_${VERSION}_linux_amd64.tar.gz && \
    tar zxf ipinfo_${VERSION}_linux_amd64.tar.gz && \
    mv ipinfo_${VERSION}_linux_amd64 /usr/bin/ipinfo && \
    rm -rf ipinfo_${VERSION}_linux_amd64.tar.gz
CMD ["ipinfo","version"]

docker container run --rm -t demo-both "hello world"  
--rm 容器退出时就能够自动清理容器内部的文件系统

docker container exec -it <ContainerID> sh 进入容器交互模式
docker container run -it -v ${pwd}:/app node  容器中的app目录和本机当前目录绑定到一起
docker container run -d -p 80:80 wordpress  映射端口
 -p 端口映射
docker container inspect --format '{{.Config.ExposedPorts}}' <ContainerID>  查看端口
docker-compose就是把很多Docker命令写入一个专属的文件docker-compose.yml，然后执行这个文件，就可以直接启动我们想要的容器
 -p就是做端口映射 80:80 服务器端口：容器端口 -d 后台运行 -it 启动交互模式
docker-compose up
docker-compose stop
docker compose ps
docker-compose.yml

version: "3.8"

services: # 容器
  servicename: # 服务名字，这个名字也是内部 bridge网络可以使用的 DNS name
    image: # 镜像的名字
    command: # 可选，如果设置，则会覆盖默认镜像里的 CMD命令
    environment: # 可选，相当于 docker run里的 --env
    volumes: # 可选，相当于docker run里的 -v
    networks: # 可选，相当于 docker run里的 --network
    ports: # 可选，相当于 docker run里的 -p
  servicename2:

volumes: # 可选，相当于 docker volume create

networks: # 可选，相当于 docker network create
